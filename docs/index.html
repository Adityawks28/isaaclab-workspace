Overview

Learn RL by building it

Everything runs in Docker, built FROM the official Isaac Sim image with Isaac Lab and RL libraries layered on top. Custom tasks live in this repo as pip-installable gymnasium plugins, so the stock training and play scripts discover them without ever editing Isaac Lab. Every experiment ships with a self-contained, interactive dashboard, so nothing stays a black box.

🐳

Zero local install

One script clones and pins Isaac Lab and builds the image, reusing warm caches.

🧩

Tasks as plugins

Registered via a gymnasium entry point; the Isaac Lab source is never touched.

🔬

Experiment-driven

Reward, lift-gate, curriculum and friction sweeps, each with an interactive dashboard.

💻

Laptop-friendly

Laptop-safe env counts; headless training, GUI eval, and a laptop↔workstation flow.

📈

Weight-independent metrics

Every comparison uses weight-independent success metrics, never the raw reward.

🎥

Watch in sim

Render any checkpoint to video, or watch the policy live in the Isaac Sim viewer.

Approach A. Isaac Lab's Docker tooling builds an image from the Isaac Sim base; our task is the thin top layer we iterate on.

Read top-down, the stack is also the pipeline: a reward becomes a PPO update becomes a moving arm.

# set up once, then train & watch
$./scripts/setup.sh # clone+pin Isaac Lab, build image, install tasks
$./scripts/train.sh --task <id> --num_envs 128 # headless PPO
$./scripts/eval.sh  --task <id> # watch in the Isaac Sim GUI

Reach

Converged
Does
Drives the arm's end-effector onto a target pose marker.
Target
A randomly-placed target position & orientation that resets every episode.
Workflow
PPO (rsl_rl) · obs = joints + target + last action · action = 7 joint targets · reward = −distance · trained headless across ~256 parallel arms.
Result
Converges reliably, about 0.12 m end-effector tracking error.
Improve
Already solid; orientation tracking could be tightened with reward re-weighting.

Lift · pick-and-place

In progress
Does
Reaches a cube, closes the gripper, and tries to lift & carry it to a goal.
Target
Grasp the cube, lift it past a height gate, deliver it to a commanded goal pose.
Workflow
PPO · obs adds the object + goal · action = 7 joints + a binary gripper · staged gated reward (reach → lift → carry) · 128 to 256 envs.
Result
Reliably reaches; lifts intermittently, rarely carries. The bottleneck is grasp reliability, not goal distance.
Improve
More parallel envs for grasp exploration (a 256-env run is training now); ultimately demonstrations / a VLA (Milestone 3).
Results

What the experiments showed

Controlled studies on the laptop (RTX 4050, 6 GB → 128 envs), fixed seed and budget, compared on weight-independent metrics.

Reach

≈ 0.12 m
  • Converges easily; a reward-weight sweep showed weights set priorities, not absolute quality.
  • Position-tracking error around 0.12 m end-to-end.

Lift

Bottleneck identified
  • A lift-gate sweep showed only the easy gate sustains lifting.
  • More iterations plateau; a goal curriculum and higher friction don't help.
  • The limiter is grasp reliability, not goal distance. More parallel envs (workstation scale) is the next lever.
