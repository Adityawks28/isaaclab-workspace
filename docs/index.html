[ 01 ] System// the build is the curriculum

Learn RL by building it

Everything runs in Docker, built FROM the official Isaac Sim image with Isaac Lab + RL libraries layered on top. Custom tasks live in this repo as pip‑installable gymnasium plugins, so the stock training and play scripts discover them without ever editing Isaac Lab.

01 / DOCKER

Zero local install

One script clones + pins Isaac Lab and builds the image, reusing warm caches.

02 / PLUGINS

Tasks as plugins

Registered via a gymnasium entry point — the Isaac Lab source is never touched.

03 / SWEEPS

Experiment-driven

Reward, lift-gate, curriculum & friction sweeps, each with an interactive dashboard.

04 / LAPTOP

6 GB-friendly

Laptop-safe env counts; headless train, GUI eval, workstation sync over Tailscale.

05 / REMOTE

Semi-autonomous

Drive the whole loop from your phone; the GPU never leaves the machine.

06 / HONEST

No cherry-picking

Weight-independent success metrics — never trust the raw reward number.

// Approach A
Isaac Lab's Docker tooling builds an image from the Isaac Sim base; our task is the thin top layer we iterate on.

Read top‑down, the stack is the pipeline: a reward becomes a PPO update becomes a moving arm.
# set up once, then train & watch
$ ./scripts/setup.sh # clone+pin Isaac Lab, build image, install tasks
$ ./scripts/train.sh --task <id> --num_envs 128 # headless PPO
$ ./scripts/eval.sh  --task <id> # watch in the Isaac Sim GUI

[ 04 ] Telemetry// what the experiments showed

Results & findings

Controlled studies on the laptop (RTX 4050, 6 GB → 128 envs), fixed seed + budget, compared on weight‑independent metrics.

Reach

≈ 0.12 m
  • Converges easily; a reward-weight sweep showed weights set priorities, not absolute quality.
  • Position-tracking error ~0.12 m end-to-end.

Lift

BOTTLENECK FOUND
  • Lift-gate sweep: only the easy gate sustains lifting.
  • More iterations plateau; a goal curriculum doesn't help.
  • Diagnosis: the limiter is grasp reliability, not goal distance — workstation-scale parallelism is next.
